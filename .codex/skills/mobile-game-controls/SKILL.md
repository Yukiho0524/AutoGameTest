# 手機遊戲操作詞彙

本 skill 是所有 Android 模擬器 agent 的共用詞彙表。讀使用者任務時，先把口語操作轉成畫面目標與 ADB 動作；每個操作後仍必須截圖驗證。

## 動詞對照

| 使用者口語 | AI 對應動作 | ADB 對應 |
|---|---|---|
| 點、點擊、按 X | 點一下 X 的中心或可點擊區域 | `input tap <x> <y>` |
| 雙擊 X | 在同一位置快速點兩下 | 連續兩次 `input tap <x> <y>`，間隔約 100-250ms |
| 長按 X、按住 X | 按住不放 | `input swipe <x> <y> <x> <y> <600-1200ms>` |
| 拖曳 X 到 Y、把 X 拖到 Y | 從 X 按住移到 Y | `input swipe <x1> <y1> <x2> <y2> <400-900ms>` |
| 往上滑、往下滑、往左滑、往右滑 | 手指快速滑動 | `input swipe <start> <end> <200-500ms>` |
| 捲動清單、捲動頁面 | 沿清單方向滑動直到看到目標 | 依目標方向多次短 swipe，逐次截圖確認 |

## 解讀規則

- 「按 X」預設是點擊，不是鍵盤按鍵；只有「按住」「長按」才使用 long press。
- 「點擊 X」要先在截圖中找到 X 的文字、圖示、按鈕或卡片中心；找不到就不要盲點。
- 「雙擊」兩次點擊後只需截圖一次驗證，除非第一次點擊已導致畫面跳轉。
- 「長按」用原地 swipe 模擬，時間先用 800ms；若遊戲需要更長才開選單，可提高到 1200ms。
- 「拖曳 X 到 Y」先定位 X 與 Y；若 X 是角色、道具、卡片，從物件中心開始拖到目標格中心。
- 「滑動」描述的是手指方向：往上滑 = 起點偏下、終點偏上；往下滑 = 起點偏上、終點偏下。
- 「向下捲動頁面」「看更多下方內容」「清單往下看」通常代表內容要往下走，因此手指要往上滑。
- 「向上捲動頁面」「回到上方」通常代表內容要往上走，因此手指要往下滑。
- 需要連續捲動時，每次只滑一小段或一頁，截圖確認是否看到目標；看到目標就停止捲動。

## 寫法範例

1. 點擊右下角便利機圖示。
2. 雙擊角色頭像放大檢視。
3. 長按道具圖示開啟快捷選單。
4. 把角色從備戰區拖曳到上陣格。
5. 在商品清單往上滑瀏覽更多商品。
6. 向下捲動好友清單到底部。

## 安全邊界

- 如果目標在付費、轉蛋、購買、登入、授權、帳號綁定或 PVP 畫面，停止並回報。
- 若同一動作連續兩次沒有產生預期畫面，不要繼續重複點擊；重新判斷畫面。
- 拖曳與長按容易誤觸，執行前要確認目標不是付費按鈕或不可逆操作。
