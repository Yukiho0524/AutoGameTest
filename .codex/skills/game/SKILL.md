# 便利商店

## 遊戲概述

《便利商店》此設定對應 Android 模擬器上的 Gamania／遊戲橘子便利商店經營遊戲，公開資料常以《PocketStore》或《便利商店口袋版》稱呼。已知公開頁面顯示其玩法核心是讓玩家化身店長，經營店鋪、配置貨架、引進商品、觀察市場趨勢、提升顧客滿意度、訓練員工，逐步打造繁榮便利商店。目標不是戰鬥，而是以低風險、可回溯的選單操作完成日常經營。

公開資料來源：
- APKCombo PocketStore：https://apkcombo.com/tw/pocketstore/com.gamania.pocketstorem/
- TapTap PocketStore：https://www.taptap.io/app/33712789
- 聯合新聞網《便利商店》歷史介紹：https://udn.com/news/story/120910/7046748
- 橘人誌 G!VOICE 開發訪談：https://brand.gamania.com/gvoice/8791/
- 鏡週刊便利商店口袋版報導：https://www.mirrormedia.mg/tag/5a6074a2160ac910002957c2

公開資料顯示 2024 年測試版曾標示「刪檔技術性測試、不開放付費功能、開放體驗至 6/10」，但實際版本與營運狀態需以模擬器當下畫面為準。若畫面出現登入、授權、付費、儲值、抽選、交易或個資要求，一律停止並回報使用者。

## 啟動流程

目標環境：
- 平台：Android
- 控制：emulator
- 模擬器：LDPlayer
- instance：0
- ADB serial：`emulator-5554`
- package：`com.gamania.pocketstorem.gama`

啟動前檢查：
1. 確認 LDPlayer instance 0 已開啟，ADB 可連線。
2. 使用 `adb -s emulator-5554 devices` 確認裝置在線。
3. 使用 `adb -s emulator-5554 shell pm list packages | findstr pocketstore` 確認套件存在。
4. 若 `com.gamania.pocketstorem.gama` 不存在，可檢查是否安裝為公開測試包名 `com.gamania.pocketstorem`，但不得自行安裝來源不明 APK。

建議啟動指令：
1. 優先使用 monkey 啟動：
   `adb -s emulator-5554 shell monkey -p com.gamania.pocketstorem.gama -c android.intent.category.LAUNCHER 1`
2. 若失敗，查詢 activity：
   `adb -s emulator-5554 shell cmd package resolve-activity --brief com.gamania.pocketstorem.gama`
3. 啟動後等待 5-10 秒，截圖驗證是否進入遊戲、更新畫面或登入畫面。

ADB 截圖注意事項：
- 每一步操作後都必須截圖驗證畫面，不符合預期就停止重判。
- 建議流程：
  `adb -s emulator-5554 shell screencap -p /sdcard/autogametest_screen.png`
  `adb -s emulator-5554 pull /sdcard/autogametest_screen.png data/artifacts/latest.png`
- 點擊前先用截圖判定按鈕位置，不可依猜測連點。
- 點擊使用：
  `adb -s emulator-5554 shell input tap <x> <y>`
- 不要使用大量連續 tap；每次 tap 後等待 1-3 秒並截圖確認。
- 若畫面解析度不同，所有座標都需依當次截圖比例重新估算，不沿用舊座標。

## UI 地圖

因目前尚無本機截圖記憶，以下為依公開資料整理的預期 UI 區域，實際以截圖 OCR／視覺判讀為準。

啟動畫面／標題畫面：
- 可能顯示《PocketStore》或便利商店相關 Logo。
- 常見按鈕：開始遊戲、Tap to Start、進入遊戲、公告、登入、同意條款。
- 若出現服務條款或隱私政策，只可協助使用者閱讀到一般資訊，不可代替使用者同意涉及帳號、授權或個資的流程。

登入／帳號畫面：
- 可能出現 Gamania、beanfun!、Gama Play 或第三方登入。
- 這是硬邊界：不代輸帳密、不處理第三方登入授權、不讀取 auth 檔。
- 遇到登入需求時停止，請使用者本人操作。

主店鋪畫面：
- 中央：便利商店平面或等角視角，顧客、貨架、收銀台、裝飾物。
- 上方：貨幣、等級、經驗、體力／能量、公告或活動入口。
- 下方或側邊：建設、商品、員工、任務、倉庫、商店、設定等主選單。
- 可安全操作：收取營收、完成日常任務、補貨、升級一般貨架、查看員工狀態。
- 需謹慎操作：消耗高價貨幣、鑽石、付費代幣、加速券、抽選券。

貨架／商品配置畫面：
- 玩法核心是配置貨架、引進熱銷商品、滿足顧客需求。
- 優先選擇免費、任務要求、低成本且可撤銷的商品配置。
- 若有市場趨勢提示，優先配置趨勢商品或顧客願望商品。
- 不為追求效率消耗付費貨幣。

員工畫面：
- 可能包含招募、訓練、派班、升級。
- 安全操作限於免費訓練、任務要求的低風險升級、已明確標示免費的派班。
- 招募若涉及抽卡、付費貨幣或機率型機制，停止並回報。

任務／成就畫面：
- 可領取免費任務獎勵與成就獎勵。
- 若領取會導向廣告、付費、登入或活動授權，停止。
- 任務文字是下一步操作的重要依據，操作前先截圖並讀取任務要求。

商城／儲值／活動畫面：
- 高風險區域。
- 包含儲值、禮包、月卡、特惠、轉蛋、抽選、購買、商城、GASH、beanfun! 支付等入口。
- 不進行購買、不確認付款、不消耗現金等價資源。

設定／信箱／公告：
- 設定可用於檢查音量、語言、帳號狀態。
- 信箱可領取明確免費獎勵；若要求登入、綁定、付款或領取高風險補償，停止。
- 公告只閱讀，不點擊外部支付、活動授權或下載連結。

## 例行任務

安全例行流程：
1. 啟動遊戲並截圖確認目前狀態。
2. 若在標題畫面，點擊「開始／進入」類按鈕，操作後截圖驗證。
3. 若進入主店鋪，先查看是否有明顯新手教學箭頭；若有，只執行教學要求的低風險點擊。
4. 收取可見的免費營收、任務獎勵或信箱獎勵，每次領取後截圖確認。
5. 查看任務列表，優先完成「補貨、配置貨架、上架商品、訓練員工、升級普通設施」等日常任務。
6. 檢查貨架或商品是否缺貨；若補貨成本為普通遊戲幣且不涉及付費貨幣，可補貨。
7. 若市場趨勢或顧客願望提示某類商品，優先用免費／普通貨幣方式上架。
8. 若有員工疲勞、訓練或派班提示，只執行免費且明確可逆的操作。
9. 完成後停在主店鋪畫面，截圖保存結果。

優先級：
- 第一優先：免費獎勵、主線／新手任務、補貨、低成本營運。
- 第二優先：普通貨幣升級貨架、開放新商品、員工訓練。
- 第三優先：裝飾、活動任務、限時任務，但只在不涉及付費與登入授權時操作。
- 禁止：儲值、購買、月卡、禮包、抽選、PVP／排行競爭、第三方登入授權。

## 風險守則

硬性規則：
- 不代輸帳密。
- 不處理第三方登入授權。
- 不代為消費。
- 不自動打線上排位。
- 每一步操作後截圖驗證。

登入與帳號：
- 遇到 Gamania、beanfun!、Gama Play、Google、Apple、Facebook、LINE 或其他第三方登入畫面，立即停止。
- 不輸入帳號、密碼、驗證碼、手機號碼、Email。
- 不讀取 app 私有資料、token、auth 檔或任何可恢復帳號狀態的檔案。
- 不代替使用者點擊「同意授權」或「綁定帳號」。

消費與付費：
- 遇到商城、儲值、GASH、月卡、禮包、限時特惠、首儲、付款確認、抽選、轉蛋等畫面，停止並回報。
- 即使按鈕看似免費，也要確認是否消耗鑽石、付費幣、票券或現金等價資源。
- 不使用付費貨幣加速、不購買商品包、不抽員工或道具。

線上與競爭：
- 本遊戲主要為經營模擬；若出現排行榜、競賽、玩家對抗、公會競爭、店鋪戰等線上競爭內容，不自動參與。
- 可閱讀排名或任務說明，但不進入會影響其他玩家或競爭分數的自動操作。

操作安全：
- 不盲目連點；每次 tap 後等待並截圖。
- 不在未知彈窗上點擊確認，先判讀文字。
- 不關閉可能重要的錯誤、維護、更新或資料異常提示，先截圖回報。
- 不更改系統設定、模擬器帳號、Google Play 帳號或付款設定。
- 若畫面與預期不符，停止重判，不靠猜測繼續。

## 圖片記憶

目前沒有已建立的圖片記憶。

已知畫面：
- 截圖路徑：無。
- signature 摘要：無。
- 可點區域：無已驗證座標。
- 風險標籤：無。

待建立記憶格式：
- 主店鋪畫面：建議標記為 `safe` 或 `routine`，記錄主選單位置、任務入口、免費獎勵入口。
- 標題畫面：建議標記為 `low`，只允許點擊開始遊戲；若含登入入口，登入區標記 `risk`。
- 登入／授權畫面：標記為 `risk`，不可建立安全自動操作。
- 商城／儲值／禮包／抽選畫面：標記為 `risk`，不可建立快速規則。
- 補貨／商品配置畫面：若確認只消耗普通遊戲幣，可標記 `routine`，但需記錄成本區域與確認按鈕文字。
- 任務獎勵畫面：若確認獎勵免費，可標記 `routine`。

圖片記憶寫入原則：
- 只保存截圖路徑、signature、畫面摘要、可點區域與風險標籤。
- 不把大型圖片或 base64 寫入 SKILL.md。
- 高風險畫面只能作為辨識與停止依據，不可建立 safe action。
- 新增快速規則前必須確認該畫面不涉及登入、付費、抽選、PVP 或個資。

## 操作流程

標準代打流程：
1. 啟動模擬器與遊戲。
2. 截圖保存目前畫面。
3. 判斷畫面類型：標題、登入、主店鋪、教學、任務、商城、錯誤或未知。
4. 若是登入、授權、付費、抽選、PVP 或未知高風險畫面，停止並回報。
5. 若是標題畫面，點擊開始／進入，截圖驗證。
6. 若是主店鋪畫面，先尋找可領取獎勵、任務提示、補貨提示。
7. 每次只執行一個動作：點擊、等待、截圖、比對結果。
8. 完成安全日常後，整理本次結果與可重用畫面資訊。
9. 若發現可穩定辨識的安全畫面，可在最終輸出 `AUTOGAMETEST_VISUAL_MEMORY` 或 `AUTOGAMETEST_FAST_RULES`，但不得包含登入、付費、抽選或線上競爭操作。

畫面判斷策略：
- 優先讀取大字標題、按鈕文字、任務文字、貨幣圖示。
- 優先點擊明確的「領取、完成、確認免費、補貨、任務、返回」。
- 對「購買、儲值、前往、抽取、十連、授權、登入、綁定、付款、確認支付」保持停止策略。
- 若按鈕文字不清楚，先放大截圖或重新截圖，不以顏色猜測。

ADB tap 策略：
- 所有座標必須從當前截圖推導。
- 優先點擊按鈕中心，不點擊邊緣。
- 避免點擊右上角付費貨幣加號、商城入口、活動浮標。
- 點擊後等待 1-3 秒；若是載入或動畫畫面，等待 5-10 秒再截圖。
- 若連續兩次操作未改變畫面，停止並重新判斷，不重複點擊同一位置。

可建立快速規則的候選：
- 主店鋪畫面上明確免費的「領取」按鈕。
- 任務完成彈窗的免費「領取」。
- 已驗證普通遊戲幣補貨按鈕。
- 教學箭頭要求的單一步驟，但僅限不涉及登入、付費、抽選、授權。

不可建立快速規則：
- 登入、授權、綁定帳號。
- 商城、儲值、月卡、禮包、抽選、付費加速。
- 排行、競賽、PVP 或玩家互動。
- 未知彈窗的確認按鈕。
- 任何可能消耗稀有付費貨幣的操作。
