# 遊戲王 Master Duel 操作技能

## 遊戲概述

Yu-Gi-Oh! Master Duel（Konami，Unity 引擎）是免費制的遊戲王 OCG/TCG 數位卡牌對戰遊戲。
核心貨幣是「寶石」（抽卡包用）。玩法為回合制卡牌對戰，AI 操作可行性高。

- 帳號：Yukiho（Lv.31+），Player ID `936-575-164`，遊戲語言：繁體中文
- 每日任務每個 40 寶石、登入每日 20 寶石（每月第 10/20 次為 50）
- 每日/登入任務每天 18:00 UTC（台灣時間凌晨 2:00）發放，月底重置

## 啟動流程（已於 2026-07-08 實測驗證）

1. **必須透過 Steam 協定啟動**（直接執行 exe 會被 DRM 擋）：
   ```powershell
   Start-Process "steam://rungameid/1449850"
   ```
2. 等待約 20~25 秒後確認程序存在：`Get-Process -Name masterduel`
3. computer-use 權限：`request_access` 要用 **"masterduel.exe"**（必須含 `.exe`；"masterduel"、"Yu-Gi-Oh! Master Duel" 都解析失敗）。程序啟動後才能請求。建議一併請求 `"steamwebhelper.exe"`（Steam 視窗有時會蓋住遊戲畫面）。
4. 若遊戲不在前景：`open_application` 用名稱 **"Masterduel"**（grant 後的 displayName，不是 masterduel.exe）。
5. 開機畫面流程：
   - 開場劇情動畫 → **點擊遊戲畫面中央 2 次**（間隔 3 秒）跳過
   - 右下角出現 `CONNECTING` → 等待約 10~15 秒
   - 標題畫面（顯示 Ver、Player ID、大字 GAME START）→ **點擊 GAME START**
   - 可能出現「資料下載」視窗（版本更新時）→ 點 OK，等下載到 100% 後點「開始」
   - 可能出現公告視窗（通知/遊戲維護/問題 三分頁）→ 點右上角圓形 X 關閉
   - 到達主選單（左側 DUEL / DECK / SOLO / SHOP 大字選單）

## 每日簽到領獎流程（每日例行，已實測）

1. 啟動遊戲並走完開機流程到主選單（見上）
2. 點 MISSION 面板（636,765）進入任務畫面
3. 點右下「全部領取」（1226,797）→ 領取登入獎勵等所有可領項
4. 「獲得物品」彈窗 → 記錄獲得內容（通常為寶石×20）→ 點 OK
5. 返回主選單，點禮物箱圖示（1130,345）檢查贈禮盒
6. 若非空 → 點「全部接收」（1226,797）；若空 → 直接返回
7. 截圖記錄主選單寶石數量作為當日成果驗證（例：2026-07-08 執行後 20→40）

## 任務系統知識

- **每日任務**：需實際對戰達成（如：發動陷阱卡 0/2、特殊召喚怪獸 0/5、在單人模式中決鬥 0/3、融合/鐘擺/連結召喚 0/3），每個 40 寶石，當月有效
- 每次登入發 3 個每日任務，最多囤 9 個
- **「在單人模式中決鬥」類任務可透過 SOLO 模式（打 AI）完成，風險最低**
- 登入任務：每日 1 次，寶石×20，月累計（如 3/31）
- DUEL LIVE：觀戰 1 場拿 5 寶石

## 風險守則（重要）

- Master Duel 是線上遊戲，Konami ToS 禁止自動化。**僅執行低頻選單操作**（登入、領獎、查看），這類操作風險低。
- **不要自動化排位（Ranked）對戰**。若要解對戰類任務，只用 SOLO 單人模式。
- 操作節奏保持人類速度：每次點擊間隔至少 1~3 秒。
- 系統維護時段無法登入（公告會顯示，如 07/09 12:00-15:30）。

## 經驗教訓（每次執行後在此追加修正）

- 2026-07-08：初次驗證。遊戲程序可能被使用者關閉，執行前務必先 `Get-Process -Name masterduel` 確認，不存在就重新啟動。
- 2026-07-08：重啟後遊戲可能不在前景（點擊會失敗並報 desktop shell frontmost），先 `open_application "Masterduel"` 帶回前景再操作。
- 2026-07-08：steamwebhelper 的視窗可能短暫蓋在遊戲上，若截圖出現大片遮罩/黑塊，等幾秒或請求 steamwebhelper.exe 權限確認狀況。
- 2026-07-08：公告視窗不是每次啟動都出現（同日第二次啟動就沒出現），流程要用「畫面判斷」而不是「固定步驟數」。
